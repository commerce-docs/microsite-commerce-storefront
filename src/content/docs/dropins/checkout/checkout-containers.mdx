The Checkout dropin includes several containers that provides the layout and functionality for the checkout process. They are designed to be flexible and customizable, allowing you to create a checkout experience that meets your business needs.

<Diagram caption="Checkout containers">![Checkout containers](@images/placeholder.webp)</Diagram>

<Callouts>

1. **ErrorBanner**: Displays error messages to the user.
1. **EstimateShipping**: Estimates shipping costs.
1. **LoginForm**: Allows the user to log in or create an account.
1. **MergedCartBanner**: Displays a banner when the cart is merged.
1. **OutOfStock**: Displays out-of-stock messages.
1. **PaymentMethods**: Displays available payment methods.
1. **PlaceOrder**: Allows the user to place an order.
1. **ServerError**: Displays server error messages.
1. **ShippingMethods**: Displays available shipping methods.

</Callouts>

## ErrorBanner configurations

The `ErrorBanner` container is designed to display error messages to the user, such as network errors, validation errors, or other issues that prevent the user from completing their purchase. You can customize the appearance of the banner, the error message, and the behavior of the banner (for example, whether it can be dismissed).

It provides the following configuration options:

## EstimateShipping configurations

The `EstimateShipping` container is designed to estimate and display shipping costs during the checkout process. You can confiure it to handle various scenarios, such as displaying free shipping and handling different tax display settings. 

It provides the following configuration options:

<OptionsTable
  compact
  options={[
    ['Option', 'Type', 'Req?', 'Description'],
    ['estimated', 'boolean', 'No', 'A boolean indicating whether the shipping cost is estimated.'],
    ['price', 'VNode', 'Yes', 'The rendered price component, which can be free shipping, the price including tax, or the price excluding tax.'],

    ['taxExcluded', 'boolean', 'No', 'A boolean indicating whether the tax is excluded from the shipping price.'],
    ['taxIncluded', 'boolean', 'No', 'A boolean indicating whether the tax is included in the shipping price.'],
  ]}
/>

## How to change configurations

In general, the steps for applying configuration options to containers is the same assuming that you're using the boilerplate code provided in the [Create your storefront tutorial](/get-started) section:

<Steps>

1. Open the file where the container is being rendered.

   For the `Checkout` container, it's the `/blocks/commerce-checkout.js` file.

1. Locate the configuration option(s) you want to change.

1. Modify an existing option or add a new one.

   The boilerplate code doesn't always show all options by default, so you may need to add them manually.

1. Save your changes and start the server. [Editorial note: What if the server is already running? Will these changes be picked up automatically?]

   ```bash
   npm start
   ```

1. Test the changes in your browser.
