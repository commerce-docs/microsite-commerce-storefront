Getting Started

Create your storefront

~20 minutes

Welcome! If you have 20 minutes, we can help you create one of the fastest storefronts on the web. Ready?

Big picture

From repo to render, this tutorial will guide you through the process of creating a new Edge Delivery Services storefront using our Commerce boilerplate template and starter content. The process involves the following steps:

Steps for creating your Edge Delivery Services storefront.
  1. Use the boilerplate template to create your storefront repo.
  2. Add the Code Sync GitHub app to your newly created repo. This app automatically redeploys the code whenever you push changes to the main branch of your storefront repo.
  3. Create a new folder on Google Drive or SharePoint and share it (via email) with the Code Sync app. This folder is where you will add all your document-based authoring content.
  4. Add the Sidekick Chrome extension and configure it to preview and publish your storefront.
  5. Start the development server and explore the boilerplate storefront.

Prerequisites

First, make sure you have the following accounts and tools set up on your local computer.

GitHub Sign into your GitHub account.
Node.js Verify your Node version (LTS).

Step-by-step

The quickest way to create a new storefront starts with our Edge Delivery Services boilerplate. This section shows you how to use the boilerplate to create your own Commerce storefront. The boilerplate template is pre-configured with all of our Commerce components and services to give you a fully-functional storefront out-of-the-box. Lets get started.

Create your code repo

  1. Navigate to aem-boilerplate-commerce.
  2. Click the Use this template button and select Create a new repository.
  3. Choose the GitHub organization (or owner) and create the repository name.
  4. Add a description and choose a visibility (we recommend public).
  5. Click the Create repository button and watch GitHub create your new repo.

What happens? When you use the template button (instead of cloning), GitHub copies all the content from the boilerplate repo (files and folders), but none of the commit history, remote links, or other version-control files. This ensures that your new storefront repository starts with a clean slate, ready for new development.

Add the AEM Code Sync app

  1. Navigate to the AEM Code Sync app and click the Configure button.
  2. Select the organization account to add the app to.
  3. From GitHubs AEM Code Sync page, in the Repository access section, select the repository you created and click the Save button.
  4. If the installation was successful, youll be redirected to a success screen on https://aem.live. You can close the browser tab and proceed.

Start the development server

You should see the first page of your boilerplate storefront at http://localhost:3000.

Choose One!
